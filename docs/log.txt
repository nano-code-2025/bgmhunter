  开发流程

  本地开发：vercel dev（同时启动 Vite + Serverless Functions）
           或 npx vite（仅前端，proxy 模式）

  部署：    git push → Vercel 自动构建 + 部署

  需要安装 vercel CLI 作为 devDependency，方便本地测试 serverless functions。

  Vercel 环境变量

  在 Vercel Dashboard 中设置：
  - DEEPSEEK_API_KEY — 仅服务端可访问（不加 VITE_ 前缀）

  .gitignore 确认

  确保包含：
  - .env.local
  - .vercel/
  - dist/

  Tailwind CSS

  当前通过 CDN 引入。这个阶段保持不变，未来如需定制主题再迁移到 PostCSS 构建。

   设计 Section 5：完整改动清单总结

  按执行顺序：

  Phase 1：目录重组

  1. 创建 src/ 目录，移入 App.tsx, index.tsx→main.tsx, types.ts, constants.ts
  2. 移入 components/, hooks/, services/ 到 src/
  3. 移入 data/ 到 src/data/
  4. 删除 constants.tsx（重复）、services/jamendoService.ts（Legacy）
  5. 更新 index.html script 路径
  6. 更新 vite.config.ts alias @ → src/
  7. 更新 tsconfig.json include

  Phase 2：API 安全层

  8. 创建 api/lib/rateLimit.ts — 内存 rate limiter
  9. 创建 api/lib/validate.ts — 请求校验
  10. 创建 api/deepseek.ts — DeepSeek 代理
  11. 创建 api/deezer/[...path].ts — Deezer CORS proxy
  12. 创建 src/services/api/deepseekClient.ts — 前端 API client
  13. 修改 src/services/aiService.ts — 调用 client 而非直接调 DeepSeek
  14. 移除 vite.config.ts 的 define 块

  Phase 3：部署配置

  15. 创建 vercel.json
  16. 更新 vite.config.ts dev proxy（加 /api/deepseek）
  17. 更新 .gitignore（加 .vercel/）
  18. package.json 加 vercel devDependency

  Phase 4：验证

  19. 本地 vite build 确认构建通过
  20. 本地 vercel dev 测试 API routes
  21. 推送到 GitHub，Vercel 部署
